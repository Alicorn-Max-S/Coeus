# CLAUDE.md


## Do

- Match the existing style of whatever file you're editing — new code should look like it belongs
- Explain what you changed and why after making edits
- Write descriptive git commit messages
- Keep solutions practical for a solo developer
- Keep code as simple as possible

## Do Not

- Add packages or dependencies without stating why
- Refactor working code unless asked
- Over-engineer — simpler is better
- Increase costs unnecessarily
- Change AI models without being requested to. 
- Add packages, models, etc, without confirming versions
